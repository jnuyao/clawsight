# AI 八大赛道发展全景：历史节点 × 趋势分析 × 职业机遇映射

> **Purpose**: tech-spectrum 的核心数据源。提供 AI 各赛道的发展历史、关键拐点和加速度评估，支撑用户画像与 AI 趋势的交叉分析。
>
> **Version**: 2026-Q1 | **Update cadence**: Quarterly

---

## 为什么需要这份文档

tech-spectrum 的核心任务是将用户画像与 AI 趋势交叉分析。但"AI 趋势"不是一个静态快照——它是一条正在加速的历史曲线。只有知道每个赛道从哪来、关键拐点在哪、加速度如何变化，才能准确判断：

- 用户的技能与哪些赛道相交 → **交汇点分析**
- 这些赛道正处于什么阶段（萌芽？爆发？成熟？）→ **机遇窗口判断**
- 用户如果现在进入，处于什么位置 → **光谱定位**

## 整体 AI 发展阶段划分

| 阶段 | 时间 | 标志事件 | 特征 |

## 赛道一：AI Agent & 工具链

**当前阶段**: 商用爆发期 | **加速度**: 🔥🔥🔥🔥🔥 | **职业机遇密度**: ⭐⭐⭐⭐⭐

| 阶段 | 时间 | 特征 |
|------|------|------|
| 概念验证期 | 2023.03-2023.06 | AutoGPT/BabyAGI 证明自主 Agent 的可能性，但可靠性极低 |
| 框架建设期 | 2023.06-2024.06 | Function Calling、LangChain/AutoGen/CrewAI 等框架涌现 |
| 能力突破期 | 2024.06-2025.03 | Computer Use、MCP、Devin 等产品级能力出现 |
| 商用爆发期 | 2025.03+ | Manus、Operator 等面向消费者/企业的 Agent 产品上市 |

### 关键里程碑

| 时间 | 事件 | 历史意义 |
|------|------|----------|
| 2022.10 | LangChain 发布 | 第一个主流 LLM 链式调用框架，定义了 Agent 编排范式 |
| 2023.03.30 | AutoGPT 发布 | GitHub 史上增长最快的项目之一，点燃全球对自主 AI Agent 的想象 |
| 2023.04.03 | BabyAGI 发布 | 简化的任务分解+优先级 Agent 架构，启发了大量后续框架 |
| 2023.06.13 | OpenAI Function Calling API | LLM 首次获得结构化工具调用能力，成为行业标准模式 |
| 2023.10 | AutoGen + CrewAI 发布 | 多 Agent 协作框架成熟，企业级场景可行性提升 |
| 2024.01 | LangGraph 发布 | 图结构 Agent 编排，解决了线性链的局限性 |
| 2024.03.12 | Devin AI 发布 | "首个 AI 软件工程师"，从 coding assistant 到 autonomous agent 的范式跃迁 |
| 2024.10.22 | Claude Computer Use (beta) | 首个通用计算机操作能力，AI 可以像人一样操作桌面 |
| 2024.11 | MCP 协议发布 | Anthropic 发布模型上下文协议，被称为"AI 的 USB"，快速被行业采纳 |
| 2024.12.11 | Google Project Mariner | Google 的浏览器 Agent 原型，确认所有主要 AI 实验室都在做 Agent |
| 2025.01.23 | OpenAI Operator | 第一个面向消费者的自主网页 Agent |
| 2025.03.06 | Manus AI | 通用型全流程自主 Agent，跨编码/浏览/数据分析 |

**对职业的影响**: 直接改变软件工程师、运维、产品经理的工作方式；催生"Agent 编排师"新角色

## 赛道二：AI-Native 开发

**当前阶段**: Vibe Coding 期 | **加速度**: 🔥🔥🔥🔥 | **职业机遇密度**: ⭐⭐⭐⭐

| 阶段 | 时间 | 特征 |
|------|------|------|
| 代码补全期 | 2021-2022 | Copilot 证明了 AI 辅助编程的价值，但仅限单行/片段补全 |
| IDE 集成期 | 2023-2024 | Cursor 等 AI-native IDE 出现，AI 从插件变成核心 |
| 自主编码期 | 2024-2025 | Claude Code、Devin 等工具实现文件级/项目级自主编码 |
| Vibe Coding 期 | 2025+ | 编程范式从"写代码"变为"描述意图"，AI 生成实现 |

### 关键里程碑

| 时间 | 事件 | 历史意义 |
|------|------|----------|
| 2020.06 | GPT-3 发布 | 175B 参数模型展现出惊人的代码生成能力 |
| 2021.06.29 | GitHub Copilot 技术预览 | 第一个主流 IDE 中的 AI 编程助手，定义了"AI 配对编程"范式 |
| 2021.08 | OpenAI Codex 发布 | 面向代码的专用 LLM，是 Copilot 的底层引擎 |
| 2022.03 | Copilot 正式发布 | $10/月订阅，一年内突破 100 万付费用户 |
| 2023.03 | Cursor IDE 发布 | 第一个 AI-native IDE，从"IDE+AI 插件"到"围绕 AI 重建 IDE" |
| 2023.03.14 | GPT-4 发布 | 竞赛编程顶尖水平，使更复杂的 AI 编码工具成为可能 |
| 2024.04 | Gemini Code Assist 企业版 | 100 万 token 上下文窗口，实现全代码库理解 |
| 2025.02.02 | Andrej Karpathy 提出"Vibe Coding" | 命名了从"写代码"到"描述意图"的范式转变 |
| 2025.02 | Claude Code 发布 | Anthropic 的 CLI Agent 编码工具，命令行原生的自主代码编辑 |

**对职业的影响**: 所有开发者都将被重新定义；"不会用 AI 编程"将成为竞争劣势

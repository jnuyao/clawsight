# AI Skill Layers: L0 → L4

> Framework for assessing and developing AI capabilities. Used by tech-compass to place users on the AI skill progression and recommend growth paths.

## Layer Definitions

### L0: AI User

**Definition**: Uses AI tools as end-user without customization.

| Dimension | Criteria |
|-----------|----------|
| Typical activities | ChatGPT conversations, Copilot suggestions, AI image generation |
| Evidence signals | Uses AI tools but doesn't configure them; copy-paste interaction |
| Knowledge depth | Knows what AI tools exist; can prompt effectively for basic tasks |
| Value creation | Personal productivity gains; faster information retrieval |

### L1: AI Integrator

**Definition**: Integrates AI tools into existing workflows and customizes them.

| Dimension | Criteria |
|-----------|----------|
| Typical activities | Custom GPTs, prompt engineering, workflow automation with AI, API integration |
| Evidence signals | Has built custom prompts/agents; uses AI APIs in projects; automates with AI |
| Knowledge depth | Understands prompt engineering; knows API patterns; can evaluate model outputs |
| Value creation | Team-level productivity; process automation; AI-enhanced products |
| Estimated time from L0 | 1-3 months with active practice |

### L2: AI Builder

**Definition**: Builds AI-powered products and systems.

| Dimension | Criteria |
|-----------|----------|
| Typical activities | Fine-tuning models, RAG systems, AI-native applications, Agent development |
| Evidence signals | Has shipped AI-powered features; fine-tuned models; built RAG pipelines |
| Knowledge depth | Understands model architectures; can evaluate and select models; knows training/inference tradeoffs |
| Value creation | New AI-powered products; competitive advantages through AI |
| Estimated time from L1 | 3-6 months with project experience |

### L3: AI Architect

**Definition**: Designs AI systems and strategies at organizational level.

| Dimension | Criteria |
|-----------|----------|
| Typical activities | AI platform design, model selection strategy, AI governance, team building |
| Evidence signals | Designed AI architecture for organization; led AI strategy; mentors AI builders |
| Knowledge depth | Deep understanding of AI landscape; can predict capability trajectories; knows limitations |
| Value creation | Organizational AI capability; strategic AI positioning; talent development |
| Estimated time from L2 | 6-12 months with leadership experience |

### L4: AI Definer

**Definition**: Defines new AI paradigms, frameworks, or research directions.

| Dimension | Criteria |
|-----------|----------|
| Typical activities | Research papers, new frameworks/protocols, industry standards, thought leadership |
| Evidence signals | Published influential work; created widely-adopted tools; speaks at top conferences |
| Knowledge depth | At the frontier of AI knowledge; shapes industry direction |
| Value creation | Industry-level impact; new capabilities; paradigm shifts |
| Estimated time from L3 | 1-3 years with research/innovation focus |

## Layer × Career Phase Matrix

Different career phases have different target layers:

| Career Phase | Target Layer | Rationale |
|-------------|-------------|-----------|
| Junior (0-3 years) | L1 → L2 | Build integration skills, start building |
| Mid (3-7 years) | L2 → L3 | Deepen building skills, start architecting |
| Senior (7-12 years) | L2 → L3 | Lead AI adoption, design systems |
| Staff+ (12+ years) | L3 → L4 | Shape strategy, define direction |
| Career changer | L0 → L1 | Start with integration, leverage domain expertise |
| Domain expert (non-tech) | L1 | AI Integrator is the highest-leverage layer for domain experts |

## Assessment Guidelines for tech-compass

When assessing a user's AI Skill Layer:

1. **Evidence-based**: Use behavioral data (GitHub, projects) over self-declaration
2. **Highest consistent layer**: Place at the layer where they consistently demonstrate criteria, not peak performance
3. **Domain-specific**: A user might be L3 in their domain but L1 in others
4. **Trajectory matters**: Note if they're actively progressing (learning signals, recent AI projects)

## Progression Recommendations

### L0 → L1 (AI User → AI Integrator)
- Learn prompt engineering fundamentals
- Build a custom GPT or AI workflow
- Integrate an AI API into a personal project
- **First step**: Complete one end-to-end AI API integration

### L1 → L2 (AI Integrator → AI Builder)
- Build a RAG system from scratch
- Fine-tune a model for a specific use case
- Ship an AI-powered feature to users
- **First step**: Build a simple RAG pipeline with your own data

### L2 → L3 (AI Builder → AI Architect)
- Design an AI system for a team/organization
- Evaluate and select models for production use cases
- Mentor others on AI development
- **First step**: Write an AI architecture decision document for your team

### L3 → L4 (AI Architect → AI Definer)
- Contribute to open-source AI tools/frameworks
- Publish insights on AI applications in your domain
- Speak at conferences about AI architecture
- **First step**: Open-source one of your AI tools or publish a technical writeup
